# ProjektSphinx: Przewodnik po systemie Git

Ten katalog zawiera dokumentację szkoleniową dotyczącą systemu kontroli wersji Git. Materiał pokrywa podstawy teoretyczne, konfigurację środowiska oraz zasady współpracy w zespole.

## Struktura dokumentacji

Dokument został podzielony na 3 rozdziały:

* **Rozdział 1**
    * Wprowadzenie do systemów kontroli wersji (VCS) i geneza Gita.
    * Omówienie zalet rozproszonej architektury (DVCS) oraz integralności danych.
    * Schemat architektury: przepływ danych między katalogiem roboczym a repozytorium.

* **Rozdział 2**
    * Pierwsze kroki: konfiguracja użytkownika (`user.name`, `user.email`) i inicjalizacja projektu.
    * Omówienie trzech stanów pliku: *Modified*, *Staged* oraz *Committed*.
    * Tabela z zestawieniem najważniejszych poleceń (`git add`, `git commit`, `git status`).

* **Rozdział 3**
    * Współpraca z wykorzystaniem zdalnych repozytoriów (GitHub, GitLab).
    * Mechanizmy synchronizacji zmian (`git push`, `git pull`) oraz klonowanie.
    * Najlepsze praktyki: praca na gałęziach i tworzenie czytelnych opisów zmian.

## Wymagania

Aby poprawnie korzystać z repozytorium, wymagane jest zainstalowanie klienta Git oraz podstawowa konfiguracja tożsamości opisana w **Rozdziale 2**.
